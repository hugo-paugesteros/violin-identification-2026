3  Is identification possible for humans ?

3.1 Method

A listening test was conducted in order to quantify to what extent humans are capable of identifying violins. As it is extremely difficult for humans to remember recordings to make comparisons, we decided to evaluate them in an easier ABX task. Listeners were first presented two recordings A and B containing two violins. Then a third recording that could contain either violin A or violin B was presented. The goal was to identify wether the violin played in X was the same that was played in A or in B.

As we also wanted to test to what extend this task was feasible, we split each test in four categories :

  1. X is the exact same recording as A or B (Same Player, Same Excerpt, Same Take)
  2. A, B and X are the same excerpt played by the same player (Same Player, Same Excerpt, Different Takes)
  3. A, B and X are different excerpts played by the same player (Same Player, Different Excerpts)
  4. A, B and X are the same excerpts played by different players (Same Player, Different Excerpts)

def parse_audio(filename):
    """
    Extracts Player, Violin, and Excerpt ignoring the take number.
    Regex logic: Matches P#, V[letter], and everything up to the final .[take].wav
    """
    match = re.search(r"(P\d+)-V([A-Z])-(.*?)\.\d+\.wav", str(filename))
    if match:
        player = match.group(1)
        violin = match.group(2)
        excerpt_code = match.group(3)

        # Translate the excerpt code
        excerpt_text = excerpt_map.get(excerpt_code, excerpt_code)

        return f"{player} | Violin {violin} | {excerpt_text}"
    return filename


# 3. Parse data into a list of dictionaries
data = yaml.safe_load(yaml_config)
rounds_data = []

for i, rnd in enumerate(data["test"]["rounds"]):
    info_A = parse_audio(rnd["a"])
    info_B = parse_audio(rnd["b"])

    # Since X has two options in your YAML, we parse both and join them
    x_options = [parse_audio(x) for x in rnd["x"]]
    info_X = "\nOR\n".join(x_options)

    rounds_data.append(
        {
            "Round": i + 1,
            "Recording A": info_A,
            "Recording B": info_B,
            "Target X (Possible Options)": info_X,
            "Category": category_map[i],
        }
    )
